The problem

Calling external APIs is one of the most common tasks in microservices. For years the Java/Spring ecosystem offered three main approaches — each with real trade-offs:

Spring 3 – 5
RestTemplate

Verbose, imperative, manual error handling. Officially deprecated in favor of WebClient.

Deprecated
Spring 5+
WebClient

Reactive-first design. Powerful, but overkill for simple synchronous GET/POST calls.

Reactive overhead
Netflix / OpenFeign
Feign

Declarative and clean — but a third-party dependency with its own annotation model.

Third-party

Spring Boot 4 brings a native, type-safe alternative: declare an interface, annotate it, and let Spring generate the implementation. No RestTemplate. No reactive chain. No extra dependency.

The old way

RestTemplate & WebClient

The imperative approach requires URL strings, manual header setup, ResponseEntity unwrapping, and explicit error handling. Even a simple GET becomes ceremony.

RestTemplate — verbose imperative 
@Service
public class TodoServiceImpl {

    private final RestTemplate restTemplate;
    private final String baseUrl = "https://jsonplaceholder.typicode.com";

    public List<Todo> findAll() {
        ResponseEntity<Todo[]> response = restTemplate.exchange(
            baseUrl + "/todos",
            HttpMethod.GET,
            null,
            Todo[].class
        );
        if (!response.getStatusCode().is2xxSuccessful()) {
            throw new RuntimeException("Failed: " + response.getStatusCode());
        }
        return Arrays.asList(response.getBody());
    }

    public Todo findById(Integer id) {
        return restTemplate.getForObject(
            baseUrl + "/todos/{id}",
            Todo.class,
            id
        );
    }

    public Todo create(Todo todo) {
        HttpHeaders headers = new HttpHeaders();
        headers.setContentType(MediaType.APPLICATION_JSON);
        HttpEntity<Todo> request = new HttpEntity<>(todo, headers);
        return restTemplate.postForObject(
            baseUrl + "/todos", request, Todo.class
        );
    }
}
WebClient — reactive chain for a synchronous call 
public List<Todo> findAll() {
    return webClient.get()
        .uri("/todos")
        .retrieve()
        .bodyToFlux(Todo.class)
        .collectList()
        .block();  // blocking in a reactive chain — defeats the purpose
}

What hurts

  • URL strings everywhere — typos become runtime errors
  • Manual deserialization — ResponseEntity unwrapping, type tokens
  • Error handling boilerplate — every call needs status checks
  • HTTP leaks into business logic — headers, methods, URIs scattered through service code
The new way

@HttpExchange interfaces

Define an interface. Annotate the methods. Spring generates the implementation at startup. You get a type-safe client with zero boilerplate.

The entire HTTP client — that's it 
@HttpExchange("/todos")
public interface TodoService {

    @GetExchange
    List<Todo> findAll();

    @GetExchange("/{id}")
    Todo findById(@PathVariable Integer id);

    @PostExchange
    Todo create(@RequestBody Todo todo);

    @DeleteExchange("/{id}")
    void delete(@PathVariable Integer id);
}

What you get

  • Type-safe — method signatures are the contract, not URL strings
  • Familiar annotations — @PathVariable, @RequestBody, @RequestHeader — same as server-side controllers
  • Zero implementation code — Spring generates the proxy from the interface
  • Testable — mock the interface like any other Spring service
Configuration

@ImportHttpServices

In earlier Spring 6 previews, wiring up an HTTP interface required manual HttpServiceProxyFactory boilerplate — creating a RestClient, wrapping it in an adapter, building the factory, calling createClient(). Spring Boot 4 eliminates all of it.

Before — manual proxy factory (Spring 6) 
@Configuration
public class HttpClientConfig {

    @Bean
    TodoService todoService() {
        RestClient client = RestClient.builder()
            .baseUrl("https://jsonplaceholder.typicode.com")
            .build();

        RestClientAdapter adapter =
            RestClientAdapter.create(client);

        HttpServiceProxyFactory factory =
            HttpServiceProxyFactory.builderFor(adapter)
                .build();

        return factory.createClient(
            TodoService.class
        );
    }
}
After — @ImportHttpServices (Spring Boot 4) 
@Configuration
@ImportHttpServices(
    types = { TodoService.class }
)
public class HttpClientConfig {

    @Bean
    RestClientHttpServiceGroupConfigurer configurer() {
        return (clientBuilder, group) -> {
            clientBuilder.baseUrl(
                "https://jsonplaceholder.typicode.com"
            );
        };
    }
}

Groups

The RestClientHttpServiceGroupConfigurer receives a group context, so you can configure multiple HTTP clients at once — shared base URL, default headers, timeouts, error handlers. Each @HttpExchange interface becomes a Spring bean automatically.

Using it

Just inject and call

The controller has no idea it's making HTTP calls. It injects TodoService like any other Spring service and calls methods on it. No HTTP details leak into business code.

A controller that calls an external API 
@RestController
@RequestMapping("/api/todos")
public class TodoController {

    private final TodoService todoService;

    public TodoController(TodoService todoService) {
        this.todoService = todoService;
    }

    @GetMapping
    public List<Todo> all() {
        return todoService.findAll();
    }

    @GetMapping("/{id}")
    public Todo byId(@PathVariable Integer id) {
        return todoService.findById(id);
    }

    @PostMapping
    public Todo create(@RequestBody Todo todo) {
        return todoService.create(todo);
    }
}

Why this matters

The controller above looks identical whether TodoService is backed by a database, an in-memory list, or an HTTP call to a remote API. The abstraction is the interface — swapping implementations is trivial, and testing requires nothing more than a mock.

Side by side

The full picture: what it took before vs. what it takes now.

Before — RestTemplate 
// Configuration
@Bean
RestTemplate restTemplate() {
    return new RestTemplateBuilder()
        .rootUri("https://api.example.com")
        .defaultHeader("Accept", "application/json")
        .setConnectTimeout(Duration.ofSeconds(5))
        .build();
}

// Usage in service
HttpHeaders headers = new HttpHeaders();
headers.setContentType(MediaType.APPLICATION_JSON);

ResponseEntity<Todo[]> response =
    restTemplate.exchange(
        "/todos",
        HttpMethod.GET,
        new HttpEntity<>(headers),
        Todo[].class
    );

if (response.getStatusCode().is2xxSuccessful()) {
    return Arrays.asList(response.getBody());
} else {
    throw new ApiException(
        response.getStatusCode()
    );
}
After — @HttpExchange 
// Interface definition
@HttpExchange("/todos")
public interface TodoService {
    @GetExchange
    List<Todo> findAll();
}

// Configuration
@Configuration
@ImportHttpServices(
    types = { TodoService.class }
)
public class HttpClientConfig {
    @Bean
    RestClientHttpServiceGroupConfigurer
    configurer() {
        return (clientBuilder, group) ->
            clientBuilder.baseUrl(
                "https://api.example.com"
            );
    }
}

// Usage — that's literally it
return todoService.findAll();
